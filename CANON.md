# SilentApply — Canon

SilentApply is released under the Apache 2.0 License.

## Purpose

SilentApply exists to reduce friction in professional coordination.

It does not optimize outcomes.
It does not create leverage.
It does not persuade.

It quietly makes common interactions easier.

---

## What SilentApply Is

SilentApply is a **quiet professional presence**.

It provides:
- a single, public profile link
- clear, candidate-controlled information
- bounded answers to common questions
- simple coordination for conversations

It is designed to feel **inevitable**, not impressive.

The ideal reaction is:
> “This is already handled.”

---

## What SilentApply Is Not

SilentApply is not:
- a job-hacking tool
- a negotiation bot
- a growth funnel
- a lead capture system
- a marketplace
- a personal brand platform

If a feature requires urgency, persuasion, or pressure to justify itself, it does not belong here.

---

## Core Principles

### Silence Is a Feature
Silence is not absence.
Silence is restraint.

SilentApply avoids:
- urgency
- excitement
- manipulation
- performance

Calm is the interface.

---

### Control Belongs to the Candidate
The candidate decides:
- what is visible
- what is shared
- what is answered
- what is downloadable
- when a conversation happens

The system never exceeds what the candidate has chosen to share.

---

### Clarity Over Cleverness
SilentApply favors:
- declarative language over persuasive language
- facts over inference
- boundaries over speculation

If something is not explicitly known, it is not answered.

---

### Infrastructure, Not Theater
SilentApply behaves like infrastructure:
- reliable
- boring
- predictable
- respectful

If the system draws attention to itself, it has failed.

---

## Quiet Links

A Quiet Link is a single public profile URL.

It:
- answers common recruiter questions
- shares proof when enabled
- allows optional coordination

It does not:
- require recruiter accounts
- perform identity theater
- track behavior across profiles
- explain its own existence

If a profile is unpublished, it does not appear.
There are no “private profile” messages.
Absence is intentional.

---

## Page Structure

Every Quiet Link follows the same order.

Above the fold:
1. Candidate name
2. Headline
3. Roles / focus
4. Location and work mode
5. Optional actions (if enabled)

Below the fold:
1. Proof and work
2. Resume (if enabled)
3. Q&A
4. Availability and booking (if enabled)

Nothing moves.
Nothing competes.
Nothing performs.

---

## Resume Philosophy

Resumes are **proof**, not persuasion.

By default:
- resumes are not downloadable
- download requires explicit candidate permission
- no resume is auto-shared

Future resume variations may exist, but:
- they are always derived from a canonical resume
- they are candidate-initiated
- they are clearly labeled as contextual
- they never replace the original

SilentApply does not rewrite facts or optimize language for manipulation.

---

## Recruiter Q&A

Recruiter questions are allowed without accounts.

The system answers only from:
- candidate-provided profile data
- the candidate’s resume
- information the candidate has explicitly chosen to share

If a question exceeds that scope, the response is brief and bounded.

Examples:
- “That isn’t available here.”
- “The candidate hasn’t shared that.”

No apologies.
No explanations.
No predictions.

Q&A exists to reduce repetition, not to simulate conversation.

---

## Booking

Booking is optional.

When enabled:
- slots may be held temporarily
- holds expire automatically
- confirmation is deliberate
- cancellation is quiet

There are no countdowns.
There is no urgency language.
There are no penalties.

Coordination should feel respectful, not transactional.

---

## Privacy and Logging

SilentApply logs only what is necessary for:
- reliability
- abuse resistance
- basic candidate awareness

It does not:
- track recruiters across profiles
- rank participants
- optimize behavior
- perform session replay

Recruiters are not leads.
Candidates are not inventory.

---

## Monetization

SilentApply is fully usable without payment.

Payment exists only to support **optional control**.

Free forever:
- Quiet Links
- core profile data
- proof links
- recruiter Q&A
- booking
- canonical resume upload and download
- visibility controls

Paid features may include:
- contextual resume variants
- advanced availability controls
- quiet, aggregate insights for candidates

Monetization never appears:
- on public profiles
- in recruiter flows
- in auth
- in booking
- in Q&A

Recruiters never know whether a candidate is paid.

---

## What This Will Not Become

SilentApply will not:
- introduce funnels
- add urgency
- score humans
- rank candidates
- sell access
- trade dignity for growth

If a change requires excitement to justify itself, it violates this canon.

---

## Governance

This document defines the behavior of SilentApply.

Any change that:
- adds pressure
- adds persuasion
- adds urgency
- adds hype
- breaks calm continuity

violates canon and must be rejected.

Canon changes are explicit, rare, and deliberate.

---

## Why a Canon Exists

SilentApply has a canon to protect its behavior over time.

Most software products drift.
They start with restraint and accumulate urgency.
They start with clarity and accumulate persuasion.
They start with intention and accumulate incentives.

The canon exists to prevent that drift.

It defines:
- what SilentApply is allowed to become
- what it must never become
- how decisions are evaluated when tradeoffs arise

The canon is not a marketing document.
It is not a roadmap.
It is not a list of features.

It is a set of behavioral constraints.

---

### What the Canon Is For

The canon exists to:
- preserve candidate dignity
- prevent pressure-based design
- keep recruiter interactions minimal, in respect of both parties time
- ensure optional monetization never leaks into core flows
- maintain calm continuity across features and time

It allows contributors to move quickly **without** re-litigating values.

If a proposal aligns with canon, it can proceed.
If it violates canon, it must be rejected.

No further justification is required.

---

### What the Canon Is Not

The canon is not:
- a growth strategy
- a positioning exercise
- a persuasion framework
- a replacement for implementation details

It does not explain *how* to build things.
It defines *whether* they should exist.

---

### How the Canon Is Used

The canon is consulted when:
- proposing new features
- adjusting existing behavior
- evaluating monetization ideas
- responding to user requests
- deciding what not to build

When tradeoffs arise, the canon takes precedence over:
- convenience
- novelty
- short-term opportunity
- external pressure

---

### Changing the Canon

The canon may change, but only deliberately.

Canon changes should be:
- explicit
- rare
- well-justified
- documented

SilentApply does not evolve by accident.

---

### Final Note

If you are reading this because you want to contribute, improve, or extend SilentApply:

Start here.

If something feels like it requires urgency, persuasion, or pressure to succeed, it likely does not belong.

The canon exists so SilentApply can remain calm—even as it grows.
