# 🔒 QUIET LINK PAGE LAYOUT CANON v1 (SilentApply)

## Purpose

The Quiet Link page exists to **communicate presence and clarity** without persuasion.

**Layout is not**

* a landing page
* a pitch deck
* a feed
* a funnel

**Prime mental model**

> “You’re already in the right place.”

---

## 1) Global Layout Principles (NON-NEGOTIABLE)

* Vertical, single-column flow
* Calm spacing
* No competing panels
* No “sections racing for attention”

**Rules**

* White space is intentional
* Nothing blinks, pulses, or animates for attention
* Every element must justify why it exists

If something feels like it’s “trying,” it’s wrong.

---

## 2) Above-the-Fold (SEALED)

This is the **only guaranteed visible area** without scrolling.

### Order (exact)

1. **Candidate Name**
2. **Headline**
3. **Roles / Focus**
4. **Location + Work Mode**
5. **Primary Actions (if enabled)**

Nothing else.

---

### 2.1 Candidate Name

* Plain text
* No badges
* No icons
* No emojis
* No honorifics

This establishes *identity*, not brand.

---

### 2.2 Headline

* One sentence
* Declarative
* No adjectives stacked

Allowed:

> “Senior Backend Engineer focused on distributed systems.”

Not allowed:

> “Results-driven, high-impact senior backend engineer…”

---

### 2.3 Roles / Focus

* Compact list
* No hierarchy
* No “best fit” language

Purpose: orientation, not positioning.

---

### 2.4 Location + Work Mode

* Calm, factual
* Example:

  > “Remote • US-based • Open to hybrid within 20 miles”

No maps. No pins. No urgency.

---

### 2.5 Primary Actions (Contextual)

Shown **only if enabled**:

* “Ask a question”
* “Book a conversation”
* “Download resume”

**Rules**

* Secondary visual weight
* Not styled as CTAs
* No color dominance
* No “or” dividers

Actions are affordances, not prompts.

---

## 3) Below-the-Fold Structure (ORDERED)

Once scrolled, content appears in this **fixed order**.

### 3.1 Proof & Work

Includes:

* Proof links
* Case studies
* Portfolio references

**Rules**

* Neutral labels
* No thumbnails screaming for clicks
* No engagement metrics

Proof exists to be referenced, not admired.

---

### 3.2 Resume (If Enabled)

* Single “Download resume” affordance
* No preview by default
* No explanation text

Resume is a document, not a section.

---

### 3.3 Q&A

* Appears as a reference block
* Each question stands alone
* No conversational threading UI

No “assistant typing…”
No back-and-forth illusion.

---

### 3.4 Availability & Booking (If Enabled)

* Calm list of slots
* No calendar dominance
* No heatmaps
* No “next available” emphasis

Time is offered, not sold.

---

## 4) Section Suppression Rules (IMPORTANT)

If a section has **no content** or is **disabled**:

* It does not render
* No placeholder
* No explanation

Absence must feel intentional.

---

## 5) Visual Hierarchy (SEALED)

### Typography

* Name > Headline > Body
* No jump cuts in size
* No dramatic weight shifts

### Color

* Neutral base
* One quiet accent at most
* No conversion colors (green/orange/red)

### Icons

* Avoid if possible
* If used: purely informational

---

## 6) What Is Explicitly Forbidden

* Testimonials
* Logos of past employers
* “Trusted by”
* Stats (“10+ years”, “Led X teams” unless factual and necessary)
* Badges
* Awards sections
* Social proof

SilentApply profiles do **not perform credibility theater**.

---

## 7) Mobile Behavior (STRICT)

* Same order as desktop
* No collapsible tricks to hide content
* Actions remain visible but not sticky

Never use sticky CTAs.

---

## 8) Footer (Minimal)

Allowed:

* Privacy
* Terms

Not allowed:

* Marketing copy
* Calls to action
* Cross-links

The page ends quietly.

---

## 9) Extensibility Guardrails

Any future feature must:

* Fit naturally into the existing order
* Justify its placement
* Not displace core identity elements

If it needs a “launch,” it doesn’t belong here.

---

## 10) Canon Summary

A Quiet Link page should feel like:

* A professional presence
* A reference surface
* A calm boundary

If a recruiter scrolls and thinks:

> “This is clear.”

You’ve succeeded.

If they think:

> “This is impressive.”

You’ve gone too far.
